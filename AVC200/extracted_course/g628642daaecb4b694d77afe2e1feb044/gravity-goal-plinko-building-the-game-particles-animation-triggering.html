Project Objective: Students will refine a 2D game design to optimize playability and refine visual design and animation.

Course Objectives:

  • Navigate Program Interface and tools. 
  • Use animation techniques to create linear experiences
  • Create storyboards, develop concepts, visual style
  • Use principles of animation
  • Use sound in production
  • Create movie clips, navigate timelines
  • Optimize performance
  • Distribute various output format

Project:

Game Requirements:

  • Game Design
    • Take notes on your game feedback and make appropriate changes to the design. This is a foundational skill for an design professional. 
    • at least 3 different goals each with different score values
    • at least 4 different Stationary Obstacles
  • Visual Design
    • Refine your sprite design. Does your sprite design work? Does it appear to be part of a coherent visual theme with a consistent visual style?
      • Theme possibilities: The theme can be subject based or a visual design based on color, shape, and texture
        • Subject Based Theme examples
          • food, fruits, Halloween, space, alien, New Mexico
        • Visual theme
          • Color Palette
          • Texture
          • Shape Language
    • Use sorting layers to properly composite your sprites.
  • Sound Design
    • Background Music
    • A sound effect for the ball bouncing
    • A different sound effect on at least TWO of the animated obstacles
    • Adjust the volume of music and sound FX to balance the soundscape of the game.
  • Game Functionality/User Interface Design
    • The player should be able to control the position of the ball spawner.
    • The UI should adhere to a Grid Design
      • https://www.youtube.com/watch?v=Q9efIv7ix1YLinks to an external site.
    • The UI should communicate the remaining time, balls, and score
  • Animation Design
    • at least 3 different animated obstacles
      • the animations should be refined according to the principles of motion design
    • At least two different animations that occur as a result of scoring. 
    • Consider the speed of the obstacle animations. Do they make sense or do they look like defaults? Don't use the defaults.
  • VFX Design
    • At least one static particle system, it could be part of the background or emitting from an obstacle. Make sure it enriches, and does not detract from the overall visual aesthetic.
    • Use either a trail renderer or a particle trail for the ball
    • Use a burst of particles that are triggered when the player scores.
  • Publish the Game
    • publish the game to itch.io either as a WebGL game or a downloadable Windows Game
