Project Objective: Students will model assets for a game using best practice workflow and asset management.  

Course Objectives

    • Use principles of animation
    • Create non-linear experiences
    • Create and use advanced buttons for user input
    • Manipulate text and related objects, variables and expressions
    • Create movie clips, navigate timelines

Project: 

Expand on your existing rolling ball prototype

    • Use Blender to create the following for your rolling ball game. Consider the theme of your game environment.
      • 2 - 3 different platform designs
        • All of the meshes should exhibit rounded corners
        • All of the meshes should have proper anchor point (origin) placement
      • 4 - 5 different environmental object designs
        • At least one environmental object needs to include animation
        • Use the AnimationSpeed.cs script to vary the animation speed amongst prefabs.
      • Replace ALL of the default primitives in the level
      • Add a 3rd zone to the level, adjust the other zones based on the feedback
    • Use the prefab workflow to manage all of your new models in Unity
    • Adjust and expand the palette of Materials in use
      • Consider materials with emission
      • Modify one Substance Community asset in Substance Sampler.
    • Adjust the lighting in the game to fit the theme

    • Show your new level design in Unity, discuss how you used the principle of Modular Design.

    • What was difficult?
    • Show the models in Blender in Solid AND Wireframe viewport mode
