    • thirdPersonWithBounce.unitypackage -drag the package into your assets folder
      • Please install the free version of DoTween, This packaged is reference by the scripts. 
      • https://assetstore.unity.com/packages/tools/animation/dotween-hotween-v2-27676
      • It will appear in the "My Assets" section of the package manager
    • _onTriggerEnterEventTag3D.cs 
      • Create a new blank gameObject. 
      • Add a collider
      • Set to "is trigger"
      • Correctly set the Tag that will trigger the event. "Player" in this case.
      • Use the event to at least do the following
        • Tell the player the race has started
        • Start the timer
    • _bumpPlayer.cs 
      • Add this script to any object that should bump the player
      • The amount of force is adjustable. Use units of "10" to adjust the bump force
      • The bumpEvent could be used to trigger sound or particle effects
    • _raceTimer.cs 
      • use the race timer to track the player's time on the course. 
      • Add the race timer to a blank game object
      • Add a UI TextMeshPro object to display the time
      • The startTimer() and stopTimer() functions can be triggered from Unity Events

Game Requirements

  • Functionality
    • A timer with display for time
    • A user interface (consider the font design, do not use the default font)
    • A properly tuned playerController
      • Consider the speed, jump height, etc. Change those values to work with your game design.
      • Be sure to add all ground platforms to a layer named "ground" 
    • Use the restartScene.cs script to allow the player to restart the scene after falling or being bumped off the platform. How will you notify the player to restart the scene? Consider the user interface. 
    • Add a win message and lose message. The lose message may need to be triggered by several different game conditions. Consider confetti particles at the finish line.
  • Visual Design
    • Color Palette
    • Consider Material design
    • Use lighting and the lighting settings to properly illuminate the scene
    • Use decorative objects to fill out the 3D space and create a stronger sense of place
    • Add some verticality to your level design. How can the verticality obscure or prepare the player for new sections of the course?
    • Spawning Obstacles, the simpleSpawner.cs has been updated. It will now turn on and off when the object is disabled and enabled. This allows for control of spawning using Unity Events. To update the script, remove the script from your game objects, delete the script from your assets, then import the new script, and reapply to the game objects.
  • Sound Design
    • Add background music to the level
    • There is a "bumpEvent" in the bump script. This could be used to trigger sound effects when the Player is bumped. 
    • Consider victory music if the player makes it to the finish line
  • Publishing
    • Title the game. Do not call it "3rdPerson Game" 
    • Build the game using WebGL settings
    • Publish the game on your Itch.IO channel
