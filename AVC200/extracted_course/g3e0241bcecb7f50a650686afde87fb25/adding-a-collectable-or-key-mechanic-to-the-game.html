Project Objective: Students will add a collectable mechanic to the game. 

Course Objectives

    • Create non-linear experiences
    • Create and use advanced buttons for user input
    • Manipulate text and related objects, variables and expressions
    • Create movie clips, navigate timelines

Project: 

  • Implement the changes from the class feedback
  • Create a collectable or key mechanic for the game (or both). The player must collect a number of objects to make the finish area accessible. 
    • Model (Blender OR Substance Modeler) and paint a collectable that fits the them of your game. Take some time to design this object. The texturing and design (lighting and particles) of the collectable should draw attention of the player
    • There must be a UI element on screen that lets the player know how many collectables they currently have and how many they need to complete the level
    • Add a sound and particle effect that is triggered when the collectable is "collected."
    • _collectionManager.cs
  • OPTIONAL: If you want to implement an animated bumper(s) in your game consider using the following script 
    • bumper-script.cs
    • The bumper lights up and animates when making contact with the player
  • OPTIONAL: If you would like the ball to stick more to moving platforms use the following script. Apply this script to the ballPlayer, then make sure the moving platform game object is tagged "movingPlatform"
    • platform-stick.cs

Revisit the Game requirements and improve on the game experience.

  • Sound
    • Ambient Sound
    • Dynamically Triggered Music track
    • Jump sound effect
  • VFX
    • Use or create an animated shader using shader graph (water, lava, etc)
    • Create 3 different particle effects to use in the game
  • Environment Design
    • >= (greater than or equal to) 4 objects modeled in Blender
    • > 3 objects modeled in Substance Modeler
      • use both the "clay" and the "primitive" workflows
    •  > 3 objects painted in Substance Painter
    • > 2 animated environment objects
    • A single object can "count" in multiple categories. Please note these are minimums, design your best game.
  • UI
    • Welcome Message + instructions
    • Win Message
    • Fall Message
    • Apply the "Escape to Quit" script to a blank game object to make it easier to quit the game
      • escapeToQuitNewInput.cs
    • restart the game script
  • Game Design
    • >= four distinct zones
    • reuse modular assets and prefabs to create a coherent world
    • Apply the squash and stretch animation principle to at least one animated game object
    • Adjust the Free look camera settings to fit the game design
    • Add a large "fall trigger" that prompts the user to restart the game


Deliverable:

Canvas: 

  • A presentation video detailing the following
    • < 3 minutes 
    • Present and explain the collectable mechanic
    • Demonstrate the UI, particle effects, and sound effect of the collectable.
    • How did you explain this new mechanic to the player?
    • How did you design your collectable? How does the design draw the attention of the player?
