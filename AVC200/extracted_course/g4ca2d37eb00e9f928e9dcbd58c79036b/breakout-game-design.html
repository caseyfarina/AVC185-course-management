Reference: Historical Context

This project in ART 200 originally started as a pong spin off, however, for reasons that mirror the historical progression from pong to breakout, it makes more sense for us to work on a one player game. We are setting up the fundamental visual appearance this week and next week we'll be working on the sound design, user interface and effects. Breakout..... the single player Pong.

Your assignment is to create your own breakout game with a visual theme. 

  1. On paper sketch three different possible themes for the game. Think about what each of the important game components would look like in the game design. I want you to sketch out three different themes on paper then choose which one seems to make sense or you're most attracted to to move forward with the design. Possible themes: machines, fruit, forest, computers, desert, or think of your own ideas. Please do not use a copyright protected IP (pokemon, transformers, avengers, mario, etc). Please check out this article and video about  Tetris Effect https://www.polygon.com/reviews/2018/11/9/18079222/tetris-effect-review-ps4-psvr . This game does an excellent job of giving each level a theme. We're only creating one level, but there is much inspiration to be found here. 
    • player
    • bricks (DOES NOT HAVE TO BE BRICK SHAPED) (5 different types)
    • moving obstacles (with animations triggered by collision)
    • background
    • walls
  2. Choose the design you like best and start creating sprite designs for these components in any manner you like. 
  • VISUAL DESIGN: Create your own sprites for each of the following. Import a spritesheet or individual sprites. Please use a coherent color palette. Decide if your sprites will be outlined or not. Do not mix outlined and non outlined sprites. You may use any graphics program to create sprites for the game. Be sure to save them in the PNG format with a transparent background. You may create the sprites in any manner (photos, illustrator, draw with pencil, paint, sculpt with clay, collage), you may not use someone else's clip art. How does the theme relate to the visual style? (I will ask you this question.)
    • Paddle
    • Ball
    • Walls (Invisible? Translucent)
    • Background (2D or 3D, is there a picture in the background? Or a 3D scene like Tetris Effect?)
    • Bricks (5 different designs, They DO NOT have to look like bricks. The shape and the function are your decision as the game designer)
      • A great tip for distributing objects evenly (or randomly in Unity) Distributing objects in space
    • Obstacles (3 different designs, must include animation trigger and script animated objects)

GAME DESIGN: Customize the game by making decisions about the following to create a dynamic experience.

  • paddle size
  • ball size
  • ball serve speed
  • paddle speed
  • number of bricks
  • number of obstacles
  • size of obstacles
  • number of brick hit points

TARGET: three minutes of gameplay

Deliverable: Please zip up the following

  • an mp4 of your game play
  • all three of your pen and paper sketches. Post these on the slack during the week as well if you're having a hard time deciding which one to choose.
